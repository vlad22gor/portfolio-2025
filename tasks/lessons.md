# Lessons

- Держать заметки краткими, без дублей и противоречий.
- Для сложных scallop-форм использовать quantized SVG-компонент вместо mask-пересечений.
- Для Figma handoff scallop: `pad` брать из padding контейнера, `shape` выводить из corner radius (`9999` -> `circle`, иначе `rectangle`).
- Для wave-divider использовать SVG stroke из чередующихся полуокружностей (`d=8`, `stroke=2`), длину квантизировать только количеством кружков без mask.
- Для текущего этапа вёрстки: desktop-only, один рабочий breakpoint `1360`; mobile breakpoint `360` добавляется отдельной задачей.
- Для desktop-навигации при route-transition резервировать scrollbar gutter на root (`scrollbar-gutter: stable both-edges`, fallback `overflow-y: scroll`), чтобы избежать горизонтального layout shift fixed-header и контента.
- `assets/` использовать только как временный буфер экспорта из Figma; прод-ассеты хранить и подключать из `public/media/...`.
- Для `header-button`: фиксированная высота `40`, top-align контента, `label container` с `padding-inline: 8px` и hug-width; у `selected` hover-анимация `gap` от `3` до `8` через motion spring. Для `wave-rail` (дефолт): рендерить одну общую `QuantizedWave` и показывать только активный сегмент через `path trim` (`stroke-dasharray`/`stroke-dashoffset`); ширина trim-сегмента считается от `label container` по правилу `ceil(width / d) - 1` (не уже `1` кружка), а сама общая rail-волна должна покрывать всю ширину трека (`cover-full`: `ceil(width / d)` без `-1`) для корректной центровки крайних табов.
- Для `wave-rail` trim-пересчёта избегать layout-чтений в каждом кадре: кэшировать rail-метрики (`ratio/offset/totalLength`) и обновлять их только на init/resize/`astro:page-load` или когда `data-wave-rail-ready=false`; при совпадении optimistic target с фактическим active на `astro:page-load` не перезапускать анимацию.
- Для выраженного bounce на крайних табах в `wave-rail` использовать симметричный overscan path (`fit='cover-bleed'`, `bleedCircles=2`): волна становится длиннее и сдвигается влево на `bleed*d`, чтобы справа/слева оставалось равное пространство для overshoot без дрейфа центра сегмента.
- В `wave-rail` hover-эффект лучше делать не через `gap` кнопки (он принудительно обнулён), а через смещение общей rail-волны CSS-переменной `--wave-rail-hover-offset`; для дизайн-компенсации можно добавить базовый сдвиг `--wave-rail-base-offset` (например, `-4`) и суммировать его с hover-offset в `transform`, а `hover-offset` сбрасывать в `sync()/astro:page-load`.
- Для persisted-header при soft navigation: для hover в `wave-rail` считать активность только по route-state (`aria-current='page'` -> `.active`) и не опираться на optimistic `data-wave-active-id`; `data-wave-active-id` использовать для trim/reconcile. После первого успешного trim держать `data-wave-rail-ready='true'` sticky, чтобы не показывать fallback-волну при кратковременной неготовности метрик.
- Для длинных `wave-rail` tab-hop использовать адаптивный delay-gate навигации `120..320ms` (по доле пути сегмента) и не стартовать `navigate()` по `RAF`; для сглаживания soft-nav включать глобальный prefetch (`prefetchAll + defaultStrategy='load'`), а для кейс-деталок ставить `data-astro-prefetch='false'`.
- Для `QScallop`/периметров использовать `QuantizedPerimeter` как runtime-движок (а `QuantizedScallop` — только wrapper для совместимости); квантацию вести по inside-fit: размер не растёт за дизайн-контейнер и округляется вниз.
- Исключение для footer: у `QScallop` использовать квантацию только по верхней грани (`rectangleEdges="top"`), нижняя и боковые грани остаются прямыми; допускается асимметричный inside-fit padding `84/72` (top/bottom), а фоновую заливку для боковых зазоров делать через `.site-footer::before` (top offset = scallop radius); motifs остаются декоративными (`aria-hidden`) и анимируются только по `pointer-hover` с profile-based spring (`slow`: `1.35/0.25`, `fast`: `1.05/0.25`).
- Для Astro `ClientRouter` любые DOM-runtime скрипты (`QuantizedWave/Perimeter`, интеракции навигации) должны быть идемпотентными и переинициализироваться на `astro:page-load`; на page-load монтировать только новые swapped-узлы (без глобального `scheduleAll()`), иначе возможен jank в анимации persisted-хедера.
- Для sans-типографики использовать variable `DM Sans` через axis-токены (`opsz`, `wght`), а отрицательный кернинг заголовков `t1`/`t1-compact` держать `-3%`.
- Для Agentation в этом проекте использовать только dev-only режим: аннотации запускаются через `npm run dev:annotate` (плюс `agentation-mcp server`), обычная разработка — `npm run dev` без toolbar.
- Для demo-блока `scallop-preview` цвет поверхности задавать на root через `--scallop-bg: var(--color-bg-darkened)`, а `.scallop-content` держать прозрачным, чтобы не перекрывать edge-shape.
- В `QuantizedPerimeter/QuantizedScallop` не задавать theme-дефолты (`--scallop-bg`, `--scallop-pad`) inline по умолчанию; передавать их только явно, чтобы классы контейнера могли переопределять токены.
- Для `QuantizedPerimeter edgePattern='totem'` использовать C1-гладкий профиль (`1 лоб = 2 cubic`) с раздельными `edge/crest` handle-length и гибридными углами: волна идёт по укороченной грани (`cornerInset`), а переходы между гранями делать через `corner-bridge` (cubic quarter-arc), чтобы убрать вытянутые углы и сохранить симметрию подхода.
- Для `Button` из Figma `12:1029` поддерживать `default`, `bordered`, `bordered-icon` с `iconSide='left'|'right'`; `hover` делать через `opacity: 0.9`, `pressed` через spring `scale: 0.95`, стрелки рендерить из `public/media/icons/button` и красить токеном через CSS mask (без отдельных hover-SVG).
- В `QuantizedPerimeter` держать hover-анимацию как single-type param animation через adapter-контракт (`resolveBaseParams/resolveHoverParams/interpolateParams/render`) и не возвращаться к inter-type morph между паттернами.
- Для hover-анимаций `QuantizedPerimeter` использовать единый `hoverMorphPreset='default'|'strong'` с pattern-specific интерпретацией: у `totem` меняются `waveAmplitudeRatio/waveTension` (с optional override через `hoverWaveAmplitudeDelta/hoverWaveTensionDelta`), у `scallop` меняется внутренний `scallopRadiusScale`; пунктирную inner-обводку `.scallop-preview-morph .scallop-content` не использовать.
